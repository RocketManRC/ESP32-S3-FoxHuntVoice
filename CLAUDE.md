# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

ESP32-S3 fox hunt voice beacon. Plays speech audio over I2S or PWM along with configurable tones for amateur radio direction finding (fox hunting). Speech can be synthesized on-device with Flite TTS or pre-generated from an external tool (e.g., Kokoro TTS) and uploaded as a WAV file.

## Build Commands

This is a PlatformIO project (using pioarduino fork of platform-espressif32). The `pio` binary is at `~/.platformio/penv/bin/pio`.

- **Build:** `pio run`
- **Upload:** `pio run --target upload`
- **Upload SPIFFS filesystem:** `pio run --target uploadfs`
- **Serial monitor:** `pio device monitor` (115200 baud)
- **Build + Upload + Monitor:** `pio run --target upload && pio device monitor`
- **Clean:** `pio run --target clean`

No test framework is configured.

## Hardware

- **Board:** ESP32-S3-DevKitC-1-N16R8 (dual-core Xtensa LX7, 8MB PSRAM)
- **I2S Pins:** BCK=GPIO21, WS=GPIO47, DOUT=GPIO38
- **PWM Pin:** GPIO40 (used with `#define OUTPUT_PWM`, single-stage RC filter: 1kΩ + 10nF)
- **PTT Pin:** GPIO2
- **USB CDC on boot** enabled for Serial over USB
- USB serial needs ~2s delay after `Serial.begin()` before output is visible
- **N16R8 GPIO restrictions:** GPIOs 26-32 (octal SPI flash) and 33-37 (octal SPI PSRAM) are unavailable

## Architecture

### Single-file design (`main.cpp`)

All logic is in `src/main.cpp`. The flow is:

1. **Boot:** Mount SPIFFS, load speech from `/speech.raw`
2. **USE_FLITE=1 (cache miss):** Synthesize with Flite TTS (~24s per phrase), concatenate parts with silence gaps, save to SPIFFS
3. **USE_FLITE=0 or cache hit:** Load PCM samples from `/speech.raw` into PSRAM (fast, ~ms)
4. **Playback loop:** Play tones (real-time generated), then play speech, wait, repeat

### Speech source (`#define USE_FLITE` toggle)

- **USE_FLITE=1 (Flite TTS):** On-device synthesis using Flite. Caches result to SPIFFS. Bump `SPEECH_VERSION` to invalidate cache when changing message text or voice settings. `DURATION_STRETCH` controls speech rate (>1.0 = slower).
- **USE_FLITE=0 (pre-generated WAV):** Loads a pre-converted WAV from SPIFFS. Use `scripts/wav2speech.py` to convert any WAV file to the required format, place output in `data/speech.raw`, then upload with `pio run --target uploadfs`. Flite library is excluded from the build (~3.2MB flash savings).

### WAV conversion script (`scripts/wav2speech.py`)

Converts WAV files (any bit depth, stereo/mono, PCM or float format) to the `speech.raw` cache format:
```
python3 scripts/wav2speech.py input.wav [-v VERSION] [-r SAMPLE_RATE] [-o OUTPUT]
```
- Defaults: `--version 12`, `--resample 16000`, `--output data/speech.raw`
- Handles PCM (format 1) and IEEE float (format 3) WAV files
- Resamples to target rate via linear interpolation

### Speech tuning defines

- `SPEECH_GAIN` — PCM volume multiplier applied after load. <1.0 = quieter, >1.0 = louder. No re-synthesis needed.
- `DURATION_STRETCH` — Flite phoneme duration scale (USE_FLITE=1 only). >1.0 = slower. Requires bumping `SPEECH_VERSION`.

### Speech caching (SPIFFS)

Speech audio lives at `/speech.raw` on SPIFFS (~8MB partition). File format:
- Header: 4 little-endian int32 [version, sample_rate, num_samples, num_channels]
- Data: raw int16 PCM samples

In Flite mode, the version field is checked on load; mismatches trigger re-synthesis. In pre-generated mode, the version field is ignored.

### Multi-part synthesis (USE_FLITE=1 only)

Speech is defined as an array of string parts. Each part is synthesized separately with `flite_text_to_wave()` and concatenated with configurable silence gaps using `concat_wave()`. This works around Flite's limited pause control.

### Audio output (`#define OUTPUT_PWM` toggle)

- **I2S mode** (default when `OUTPUT_PWM` not defined): Uses PCM5102 DAC via `I2SStream` from arduino-audio-tools. Tones are sine waves generated as PCM samples.
- **PWM mode** (`OUTPUT_PWM` defined): Uses `SimplePWMAudio` class — ESP32 LEDC at 156kHz/8-bit with busy-wait sample timing. Tones are square waves via direct GPIO toggling. Needs RC low-pass filter on output pin.

### Tone generation

`playTone(durationMs)` and `playSilence(durationMs)` generate tones. In I2S mode, sine wave samples are written in 256-sample chunks. In PWM mode, a square wave is generated by direct GPIO toggling (LEDC is detached during tones and re-attached for speech).

### LED status (RGB built-in, runs on Core 0)

- **Red blink:** Synthesizing speech (Flite mode, no cache found)
- **Blue blink:** Loading speech from SPIFFS
- **Green blink:** Waiting between playback cycles
- **Off:** Playing audio

### Key Libraries

- **[arduino-audio-tools](https://github.com/pschatzmann/arduino-audio-tools):** Stream-based audio processing (I2S output)
- **[arduino-flite](https://github.com/pschatzmann/arduino-flite):** Flite TTS engine (cmu_us_slt clustergen voice, ~3.2MB flash) — only linked when USE_FLITE=1

### Important Notes

- PSRAM is enabled via build flag (`-DBOARD_HAS_PSRAM`) — required for Flite and sample buffers
- Speech samples are allocated in PSRAM via `ps_malloc()`
- Build uses `-O3` optimization
- `board_build.filesystem = spiffs` is set in `platformio.ini` (pioarduino defaults to LittleFS)
- SPIFFS.begin(true) auto-formats on first mount
- When USE_FLITE=0, a minimal cst_wave-compatible struct is defined so loading and playback code works unchanged
